# portal-home-lab

一个用于归档和发布多个独立 HTML 页面的仓库。  
本仓库不绑定任何特定域名或网站，仅作为静态 HTML 页面的集中存放与版本管理用途。

## 项目简介

本仓库主要用于收集、整理和托管多个独立的 HTML 页面。  
这些页面可以是个人工具页、信息展示页、实验性前端页面或其他轻量级 Web 资源。  
仓库本身不提供任何网站导航、搜索引擎或商业服务。

## 目录说明

```
portal-home-lab/
├── pages/          # 存放独立的 HTML 页面文件
├── assets/         # 公共资源文件（CSS、JS、图片等）
├── index.html      # 入口页面（可选，用于展示页面列表）
└── README.md       # 本文件
```

- `pages/`：每个子目录或文件代表一个独立的 HTML 页面，可包含其依赖资源。
- `assets/`：供多个页面共享的样式、脚本或媒体文件。
- `index.html`：可选入口页面，通常用于列出仓库内所有页面的链接。

## 页面归档说明

- 每个页面应尽量自包含，避免依赖外部不可控资源。
- 页面命名建议简洁、有意义，避免使用特殊字符。
- 如需引用仓库内公共资源，应使用相对路径。
- 归档的页面不会主动更新或维护，除非提交者提出变更。

## 使用方式

1. 克隆本仓库到本地。
2. 在 `pages/` 目录下添加你的 HTML 文件及相关资源。
3. 可选：修改 `index.html` 以更新页面列表。
4. 提交 Pull Request 或直接推送至仓库。

## 维护说明

- 本仓库由维护者不定期整理，不保证所有页面持续可用。
- 页面内容由贡献者自行负责，仓库不承担内容审核义务。
- 如发现链接失效、资源缺失或安全问题，欢迎提交 Issue 或 PR。

## 贡献指南

欢迎提交新的 HTML 页面或改进现有页面。  
提交前请确保：

- 页面不包含恶意代码、跟踪脚本或侵犯他人权益的内容。
- 页面文件放置在正确的目录下，资源引用路径正确。
- 如有必要，更新 `index.html` 中的页面列表。
